# TMDB Movie Quality Classification & Plot Similarity Analysis

This notebook demonstrates a machine learning pipeline for classifying movie quality and finding similar movies based on their plots and metadata. It leverages Natural Language Processing (NLP) with LSTM networks for plot analysis and integrates numerical and categorical metadata.

## Project Overview

1.  **Movie Quality Classification**: Develops a deep learning model (LSTM with metadata) to classify movies into quality categories (Poor, Average, Good, Excellent) based on their `vote_average`.
2.  **Plot Similarity Analysis**: Utilizes embeddings generated by the trained model to find movies with similar plot descriptions.

## Dataset

The project uses a `movies.csv` dataset (presumably from TMDB or a similar source) containing various movie attributes.

**Key Features Used**:
*   `title`: Movie title
*   `overview`: Plot summary
*   `runtime`: Movie duration in minutes
*   `budget`: Movie production budget
*   `genres`: Movie genres
*   `original_language`: Original language of the movie
*   `vote_average`: Average user vote (used for quality labeling)

## Methodology

### 1. Data Loading and Preprocessing
*   Loads the `movies.csv` file into a Pandas DataFrame.
*   Handles missing values:
    *   `title`, `overview`, `genres` are filled with empty strings or 'Unknown'.
    *   `original_language` is filled with the mode.
*   Filters movies based on `runtime` (20 to 500 minutes).

### 2. Target Engineering
*   A `quality_label` function is applied to `vote_average` to create a categorical `quality` column:
    *   `0`: Poor (vote_average < 5)
    *   `1`: Average (5 <= vote_average < 6)
    *   `2`: Good (6 <= vote_average < 7.5)
    *   `3`: Excellent (vote_average >= 7.5)

### 3. Exploratory Data Analysis (EDA)
*   Visualizes the distribution of movie quality classes.
*   Analyzes the relationship between plot length (`text_len`) and movie quality.

### 4. NLP Processing (Text Data)
*   Combines `title` and `overview` into a single `text` feature.
*   Uses `Tokenizer` to convert text into sequences of word IDs, handling out-of-vocabulary (OOV) words.
*   Pads sequences to a fixed length of 300 using `pad_sequences`.

### 5. Metadata Processing (Numerical & Categorical Data)
*   **Numerical Features**: `runtime` and `budget` are scaled using `StandardScaler`. `budget` is log-transformed (`np.log1p`) to handle its skewed distribution.
*   **Categorical Features**: `genres` are one-hot encoded using `MultiLabelBinarizer` as movies can have multiple genres.
*   These processed features are concatenated to form `X_meta`.

### 6. Train-Test Split
*   The dataset is split into training and testing sets (80% train, 20% test) using `train_test_split`.
*   The `quality` labels are one-hot encoded using `to_categorical`.

### 7. LSTM Model Architecture
*   A deep learning model is built using TensorFlow/Keras with two inputs:
    *   **Text Input**: An `Embedding` layer followed by an `LSTM` layer (96 units) and a `Dense` layer (64 units, ReLU activation) with Dropout for regularization.
    *   **Metadata Input**: A `Dense` layer (32 units, ReLU activation) for the numerical and one-hot encoded genre data.
*   The outputs of the text and metadata branches are `Concatenated`.
*   A final `Dense` layer (64 units, ReLU) processes the combined features.
*   The output layer is a `Dense` layer with 4 units (for 4 quality classes) and a `softmax` activation.
*   The model is compiled with the `adam` optimizer, `categorical_crossentropy` loss, and `accuracy` metric.

### 8. Training and Evaluation
*   The model is trained for 7 epochs with a `batch_size` of 96 and a `validation_split` of 0.1.
*   Evaluates the model's performance on the test set using a `classification_report` and a `confusion_matrix` heatmap.

### 9. Plot Embeddings & Similarity Search
*   An `embedding_model` is created to extract the combined feature embeddings from the trained model's output before the final classification layer.
*   A `title_to_idx` mapping is created for quick lookup of movie indices.
*   A `similar_movies` function uses `cosine_similarity` on these embeddings to find the `k` most similar movies to a given title.

### 10. Prediction Function
*   A `predict_movie_quality` function takes movie title, overview, runtime, budget, and genres as input.
*   It preprocesses the input data similarly to the training pipeline.
*   Uses the trained model to predict the quality class (Poor, Average, Good, Excellent) for a new, unseen movie.

## Example Usage

The notebook includes examples demonstrating:
*   Predicting the quality of several well-known movies (e.g., "The Godfather", "Inception", "The Shawshank Redemption", "CATS (2019)").
*   Finding similar movies using the `similar_movies` function (e.g., for "The Godfather").
